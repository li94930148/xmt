# 版本发布指南

## 快速发布

运行以下命令，按照提示操作即可：

```bash
npm run release
```

## 交互式发布流程

脚本会引导你完成以下步骤：

### 1. 选择版本更新类型

```
请选择版本更新类型:
  1. Patch (1.0.1 → 1.0.2) - Bug 修复
  2. Minor (1.0.1 → 1.1.0) - 新功能
  3. Major (1.0.1 → 2.0.0) - 重大更新
  4. 自定义版本号
```

- **Patch**: 修复 Bug、小改动
- **Minor**: 新增功能、不影响现有功能
- **Major**: 重大更新、可能有不兼容变更

### 2. 输入更新标题

```
请输入本次更新标题 (如: 新增用户管理功能):
```

简洁描述本次更新的主要内容。

### 3. 输入更新内容

```
📝 请输入更新内容 (每条一行，输入空行结束):

更新内容 #1 (空行结束): 新增用户批量导入功能
请选择类型:
  1. 新增
  2. 优化
  3. 修复
  4. 安全
请选择 (输入数字): 1
✅ 已添加: [新增] 新增用户批量导入功能

更新内容 #2 (空行结束): 修复登录页面样式问题
请选择类型:
  1. 新增
  2. 优化
  3. 修复
  4. 安全
请选择 (输入数字): 3
✅ 已添加: [修复] 修复登录页面样式问题

更新内容 #3 (空行结束): [直接回车结束输入]
```

### 4. 确认发布

```
==================================================
📦 版本: v1.1.0
📋 标题: 用户管理功能增强
📝 更新内容:
   [新增] 新增用户批量导入功能
   [修复] 修复登录页面样式问题
==================================================

确认发布? (y/n): y
```

### 5. 自动完成

```
✅ 已更新 package.json
✅ 已更新 src/data/changelog.ts

🎉 版本 v1.1.0 发布准备完成！

下一步:
  1. 运行 npm run build 构建项目
  2. 部署到服务器
  3. 用户登录时会自动看到更新提示
```

## 发布后用户端效果

### 1. 更新提示弹窗
用户登录时，如果版本号与上次登录时不同，会显示更新提示弹窗：
- 显示新版本号
- 显示更新标题和主要内容
- 可点击"查看完整更新日志"跳转

### 2. 系统更新说明页面
在 **系统设置 → 系统更新说明** 可以查看完整的更新历史。

## 手动更新方式

如果不想使用脚本，也可以手动更新：

### 1. 修改版本号
编辑 `package.json`：
```json
{
  "version": "1.1.0"
}
```

### 2. 添加更新日志
编辑 `src/data/changelog.ts`，在数组顶部添加：
```typescript
{
  version: '1.1.0',
  date: '2026-06-05',
  title: '用户管理功能增强',
  changes: [
    {
      type: 'feature',
      description: '新增用户批量导入功能',
    },
    {
      type: 'fix',
      description: '修复登录页面样式问题',
    },
  ],
},
```

### 3. 构建部署
```bash
npm run build
# 部署到服务器
```

## 变更类型说明

| 类型 | 标签 | 颜色 | 适用场景 |
|------|------|------|----------|
| `feature` | 新增 | 🟢 绿色 | 新功能、新特性 |
| `improvement` | 优化 | 🔵 蓝色 | 性能优化、体验改进 |
| `fix` | 修复 | 🟡 黄色 | Bug 修复 |
| `security` | 安全 | 🔴 红色 | 安全性增强、漏洞修复 |
