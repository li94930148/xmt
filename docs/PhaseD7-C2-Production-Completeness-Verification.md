- 浏览器：Codex 内置浏览器，独立已登录会话
- 测试账号：既有管理员测试账号（未记录密码）
- 测试 Production：ID `85`
- 协作房间：`production:85`
- 验收方式：真实浏览器页面操作、重新打开记录核对，以及控制台观察。

本阶段未修改源代码、数据库、接口、权限、workflow 或 Runtime。测试写入仅通过现有 Production 编辑器的正常保存链路完成，并且只作用于 C1 可回滚测试记录。

| 用例 | 结果 | 真实浏览器证据 |
| --- | --- | --- |
| 1. 保存失败阻止离开 | 未覆盖（浏览器策略阻断） | 已准备失败保存场景；在切换离线模拟后，浏览器自动化安全策略在点击返回前拒绝该操作。未绕过或替代该策略，因此无法验证 `not_durable` / `interrupted` 时是否阻止导航、是否显示未保存提示，以及页面是否保持。|

## 3. 控制台与保存观察

- 重复返回用例的重新打开结果包含 `D7_C2_REPEAT_RETURN_TEST`，确认内容已持久化。
- 已检查页面 console 的 error / warning 列表，未观察到 XMT 应用级错误或警告。
- 本轮未为页面增加日志、探针或网络拦截；因此不把不可见的内部调用次数写成已验证结论。

## 4. 未覆盖原因

1. 保存失败用例需要在不改变业务代码和数据库的前提下模拟 `persist` 失败。浏览器离线模拟后，测试动作被浏览器自动化 URL 安全策略拦截；遵守该策略，没有尝试替代浏览器、CDP 绕过、脚本注入或直接中断服务。
2. 无 Runtime handle 需要一个真实且不改变权限、workflow 或页面挂载条件的入口。Production 85 没有这样的现成状态；本阶段不通过篡改页面运行时状态来伪造验收。
3. `gracefulDispose` 的并发复用没有 UI 或遥测可直接读取。双击结果只能证明一次最终路由与内容持久化，不能代替内部调用计数证据。

## 5. 是否建议进入 Shooting

不建议进入 Shooting 页面 graceful-dispose 接入。

Production 的重复返回已取得正向黑盒结果，但保存失败阻止离开与无 Runtime handle 兼容性仍没有真实浏览器证据，且无法通过当前允许范围安全补齐。应先提供受控、可观察的失败保存测试入口，以及不改变业务状态的 handle 缺失测试入口；完成这两项后再评估 Shooting 接入。
