# UI设计规范

## 文档说明

本规范基于当前真实页面、组件、样式入口和项目定位整理，同时补充未来“自有品牌中台”与“行业通用系统”方向下的视觉要求。

本规范强调两条线：

1. 当前真实现状
2. 未来统一升级方向

## 1. 产品视觉定位

### 当前定位

XMT 当前视觉定位已经明显偏向：

1. 现代 SaaS 中后台
2. 内容生产工作台
3. 专业、稳定、偏工具型
4. 深色优先、浅色兼容

### 中期定位

未来升级为品牌中台后，视觉应进一步统一为：

1. 专业
2. 高级
3. 稳定
4. 可识别
5. 适合长期使用

### 长期定位

如果扩展为行业通用系统，视觉语言应具备：

1. 可品牌化
2. 可主题化
3. 可配置
4. 可复用

## 2. 整体设计风格

### 当前现状

当前页面真实表现为：

1. 卡片化布局
2. 玻璃感与轻模糊效果
3. 深色主题较强
4. 编辑器场景与中后台场景并存
5. 登录页具有更明显品牌展示倾向

### 统一方向

整体设计风格统一为：

1. 现代 SaaS
2. 简洁
3. 专业
4. 高级
5. 适合传媒内容生产中台
6. 后续可自然升级为协同编辑平台

## 3. 页面布局规范

### 标准布局

当前和后续建议统一为：

1. 左侧导航
2. 顶部操作栏
3. 主内容容器
4. 页面标题区
5. 页面内容区

### 页面类型模板

建议统一四类页面模板：

1. 列表页
   - 标题区
   - 操作区
   - 筛选区
   - 列表区
   - 分页区

2. 详情页
   - 返回与标题区
   - 状态与元信息区
   - 主内容区
   - 侧边辅助区

3. 仪表盘页
   - 欢迎区
   - 指标区
   - 业务卡片区

4. 设置页
   - 分类导航
   - 配置表单区
   - 保存操作区

## 4. 颜色体系

### 当前基础色倾向

当前系统已经形成如下主色倾向：

1. 主品牌蓝
2. 深色背景层级
3. 浅色兼容层级
4. 成功 / 警告 / 危险 / 信息状态色

### 规范要求

1. 品牌色只承担关键强调和主要操作
2. 状态色优先用于流程状态、反馈和标签
3. 不在页面中随意新增独立品牌色
4. 后续逐步减少硬编码颜色，统一到主题变量

### 品牌中台阶段方向

需要逐步支持：

1. 主品牌色
2. 辅助品牌色
3. 登录页品牌展示色
4. 中后台操作色

## 5. 字体体系

### 当前现状

当前系统以 `Inter` 为主字体，搭配系统字体回退。

### 规范要求

1. 页面主标题
   - 用于页面一级标题
   - 强调清晰和稳定

2. 区块标题
   - 用于卡片和内容分区
   - 强调层级，不追求花哨

3. 正文
   - 适合长时间阅读
   - 适合表格和管理界面

4. 辅助文本
   - 用于说明、时间、提示
   - 颜色应弱于正文

## 6. 间距体系

以 4px 为基础步进，统一控制：

1. 组件内部小间距：8px
2. 表单项与列表项常规间距：12px
3. 卡片内边距：16px 到 24px
4. 页面区块间距：24px
5. 大区块分隔：32px

## 7. 圆角与阴影规范

### 圆角

建议统一：

1. 基础控件：`rounded-lg`
2. 卡片：`rounded-xl` 或 `rounded-2xl`
3. 弹窗：`rounded-2xl`

### 阴影

建议统一：

1. 普通卡片：轻阴影
2. 悬浮层：中阴影
3. 弹窗与重要浮层：强阴影

## 8. 表单规范

1. 标签统一在输入项上方
2. 必填项要明确标记
3. 焦点态使用统一品牌高亮
4. 错误态使用统一危险色
5. 长文本输入与编辑器要明显区分于普通输入框

## 9. 表格规范

1. 表头与内容区层级清晰
2. 行 hover 反馈轻量
3. 操作列尽量统一放在右侧
4. 宽表格使用横向滚动容器
5. 状态列优先使用统一标签组件

## 10. 弹窗规范

1. 所有弹窗统一遮罩层逻辑
2. 标题、说明、内容、操作区结构统一
3. 删除类弹窗使用危险语义
4. 表单类弹窗支持滚动和移动端适配

## 11. 状态标签规范

1. 流程状态统一使用一套状态标签语义
2. 成就、积分、辅助标签沿用同一视觉体系
3. 不允许各页面自由发明新的状态标签样式

## 12. 空状态、加载状态、错误状态规范

### 空状态

必须包含：

1. 图标
2. 标题
3. 说明
4. 可选操作按钮

### 加载状态

1. 优先骨架屏
2. 页面级整屏加载应尽量少用
3. 路由级加载保持轻量

### 错误状态

1. 明确错误说明
2. 提供重试或返回建议
3. 全局错误继续由错误边界兜底

## 13. 后续 UI 升级方向

### 品牌中台阶段

1. 统一登录页与后台主界面的品牌语言
2. 提供系统品牌配置能力
3. 收敛页面级硬编码视觉
4. 沉淀统一设计变量

### 协同编辑平台阶段

1. 强化编辑器场景的沉浸感
2. 强化评论、批注、版本切换视觉层级
3. 强化多人协作反馈

### 行业通用系统阶段

1. 支持主题切换
2. 支持品牌模板
3. 支持模块级视觉裁剪
4. 支持不同行业场景复用

## 结论

XMT 当前已经具备现代 SaaS 内容中台的基础气质。后续不应推翻重做视觉，而应在现有基础上完成：

1. 统一
2. 品牌化
3. 配置化
4. 平台化
