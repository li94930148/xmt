# UI设计规范

## 文档说明

本文档基于当前前端真实页面、组件、样式入口和现有项目定位整理。

- 已实现：可从当前页面、样式、布局和组件中直接确认的内容
- 规划中：结合产品定位和后续演进方向给出的统一化建议

本文档不代表当前项目已经完整拥有设计系统，只用于为后续 UI 治理和协同平台升级建立统一标准。

## 1. 产品视觉定位

### 已实现

XMT 当前视觉定位已经明显偏向以下方向：

1. 现代 SaaS 后台
2. 内容生产工作台
3. 深色优先、浅色兼容
4. 卡片化、信息面板化
5. 兼顾“管理后台”和“创作空间”

从当前页面结构看，系统不是偏营销官网风格，而是偏高频使用型工作界面，强调：

- 信息层级清晰
- 操作区稳定
- 状态反馈明确
- 页面长时间使用下的可读性

### 规划中

后续应把视觉定位进一步收敛为：

- 专业
- 高级
- 简洁
- 稳定
- 适合传媒内容生产中台
- 可自然升级为协同编辑平台

## 2. 整体设计风格

### 已实现

当前系统已呈现以下风格特征：

1. 后台主界面采用卡片式布局
2. 大量使用柔和渐变、高斯模糊和轻玻璃态效果
3. 顶部栏、侧边栏、主内容区边界明确
4. 深色模式中使用低饱和背景和高对比文本
5. 关键操作用高亮品牌色或渐变按钮强调

从具体代码可见：

- 主后台是“稳重的深色 SaaS”
- 登录页则有更强的品牌化和展示性
- 编辑器区域更偏“创作工具”视觉，而不是普通表单页

### 设计风格原则

建议后续统一遵循：

1. 后台页面优先稳重，不做过度装饰
2. 数据和流程页面强调清晰和效率
3. 创作与编辑页面允许更强的沉浸感
4. 登录页可保留更高表现力，但应与后台主风格形成同品牌体系

## 3. 页面布局规范

### 已实现

当前页面主布局可概括为：

1. 左侧固定导航
- 侧边栏用于承载主业务入口
- 支持折叠与移动端抽屉式展开

2. 顶部操作栏
- 承载搜索入口、消息入口、主题切换、用户菜单

3. 主内容区
- 使用统一内容宽度容器
- 通过面包屑 + 模块内容形成页面主体

4. 内容组织方式
- 列表页：工具栏 + 表格/卡片 + 分页
- 仪表盘页：多卡片、多分区布局
- 详情页：头部元信息 + 主编辑/展示区 + 辅助侧栏

### 规范建议

1. 列表页结构
- 页面标题区
- 筛选/搜索区
- 主数据区
- 分页区

2. 详情页结构
- 返回与标题区
- 状态与元信息区
- 主内容区
- 侧边辅助区

3. 仪表盘结构
- 欢迎区
- 核心指标区
- 业务分区卡片区

4. 移动端原则
- 优先保证导航、标题、核心操作、主要内容可用
- 次要统计与装饰可弱化

## 4. 颜色体系

### 已实现

当前项目已经形成较明确的色彩体系，主要来自 `src/index.css` 和 `src/constants/index.ts`。

#### 主题基础色

深色模式：

- 主背景：`#0f1117`
- 次背景：`#161822`
- 三级背景：`#1e2030`
- 浮层背景：`#252840`
- 主文字：`#e8eaed`
- 次文字：`#9aa0b0`
- 弱文字：`#636983`
- 边框：`#2a2d3e`
- 品牌强调：`#5c7cfa`

浅色模式：

- 主背景：`#ffffff`
- 次背景：`#f8f9fa`
- 三级背景：`#f1f3f5`
- 主文字：`#1a1d2e`
- 次文字：`#5f6672`
- 弱文字：`#9aa0b0`
- 边框：`#e5e7eb`
- 品牌强调：`#4263eb`

#### 功能状态色

- 成功：绿色
- 警告：黄色/橙色
- 危险：红色
- 信息：蓝色

#### 流程状态色

当前选题流程已形成真实状态色映射：

- `pending`
- `approved`
- `rejected`
- `production`
- `shooting`
- `publishing`
- `completed`

### 规范建议

1. 品牌色只承担主操作和关键信息强调
2. 状态色优先用于流程、反馈和标签，不大面积铺底
3. 同一状态在标签、图标、按钮提示中应保持一致
4. 后续减少页面内随意硬编码颜色，向主题变量收敛

## 5. 字体体系

### 已实现

当前系统主字体为：

- `Inter`
- 后备为系统字体栈

使用特征：

- 标题偏粗
- 正文偏中性
- 辅助说明偏细偏浅
- 大量使用小号大写标签和 tracking 扩展做信息分层

### 规范建议

1. 一级标题
- 页面主标题
- 建议使用 `text-2xl` 到 `text-3xl`
- 字重 `700`

2. 二级标题
- 卡片标题、区块标题
- 建议使用 `text-base` 到 `text-lg`
- 字重 `600`

3. 正文
- 常规内容和表格文本
- 建议 `text-sm` 到 `text-base`

4. 辅助文本
- 时间、说明、标签前缀
- 建议 `text-xs` 到 `text-sm`

5. 数据数字
- 统计卡数字建议加大并压缩字距

## 6. 间距体系

### 已实现

当前项目已经大致形成以 4px 为基础单位的间距习惯，常见为：

- 4
- 8
- 12
- 16
- 20
- 24

在 Tailwind 类名上表现为：

- `gap-2`
- `gap-3`
- `gap-4`
- `p-4`
- `p-5`
- `p-6`

### 规范建议

1. 组件内部小间距：8px
2. 表单项和列表项常规间距：12px
3. 卡片内边距：16px 到 24px
4. 页面模块间距：24px
5. 复杂页面分区间距：24px 到 32px

## 7. 圆角与阴影规范

### 已实现

当前圆角使用较统一，真实项目中常见：

- `rounded-lg`
- `rounded-xl`
- `rounded-2xl`
- 登录页存在更大圆角卡片

阴影使用特征：

- 后台页面阴影偏轻
- 重点卡片和渐变块使用更明显阴影
- 登录页和浮层使用更强投影

### 规范建议

1. 基础控件
- 使用 `rounded-lg` 或 `rounded-xl`

2. 卡片
- 使用 `rounded-2xl`

3. 弹窗
- 使用 `rounded-2xl`

4. 阴影层级
- 普通卡片：轻阴影
- 悬浮态：中阴影
- 弹窗/浮层：强阴影

## 8. 表单规范

### 已实现

当前表单真实特征：

1. 输入框多使用圆角 + 边框 + 背景层级区分
2. 深浅色下有不同输入背景
3. 焦点态普遍使用品牌色 ring
4. 表单在列表页弹窗、登录页、设置页、详情页均有不同密度表现

### 规范建议

1. 标签位置
- 统一放在输入框上方

2. 必填项
- 明确标注 `*`

3. 输入框高度
- 常规输入高度保持一致

4. 焦点反馈
- 使用统一品牌色高亮

5. 错误反馈
- 使用红色文本和边框提示

6. 长文本输入
- 使用更宽松的内边距

## 9. 表格规范

### 已实现

当前表格主要出现在：

- `Topics.tsx`
- 其他管理类和列表类页面

真实特征：

1. 表头采用弱背景色区分
2. 行间用边框分隔
3. Hover 行高亮
4. 支持加载骨架和空状态
5. 表格上方通常配搜索和筛选区

### 规范建议

1. 表头使用较弱背景，与内容区分层
2. 表头文本使用小号、加粗、弱色
3. 行 hover 只做轻微背景变化
4. 表格操作按钮统一放最右列
5. 表格过宽时统一使用横向滚动容器

## 10. 弹窗规范

### 已实现

当前弹窗主要存在于：

- 登录首次改密
- 删除确认
- 新建选题弹层
- 各类临时确认操作

真实特征：

1. 使用全屏遮罩
2. 遮罩多为半透明黑色
3. 弹层本体偏卡片式
4. 操作区按钮左右并列或右对齐

### 规范建议

1. 弹窗层级
- 遮罩 > 弹窗 > 内部操作

2. 弹窗结构
- 标题
- 说明
- 内容区
- 操作区

3. 危险弹窗
- 删除类操作使用红色确认按钮

4. 长表单弹窗
- 需支持滚动

## 11. 状态标签规范

### 已实现

当前已有真实状态标签体系：

- `StatusBadge`
- `RarityBadge`
- `PointBadge`
- 常见状态色映射

项目内状态标签特征：

1. 小尺寸
2. 圆角
3. 带边框
4. 低透明度背景
5. 文本与状态色一致
6. 部分标签带小圆点

### 规范建议

1. 流程状态统一使用一套状态标签样式
2. 成就、积分等辅助标签保持同一视觉语言
3. 避免新页面自行发明独立标签样式

## 12. 空状态、加载状态、错误状态规范

### 已实现

真实存在的状态组件和模式包括：

1. 空状态
- `EmptyState`
- `EmptyTopics`
- `EmptyData`
- `EmptySearch`

2. 加载状态
- `SkeletonLine`
- `SkeletonAvatar`
- `SkeletonCard`
- `SkeletonTable`
- 路由懒加载转圈态

3. 错误状态
- `ErrorBoundary`
- 通知提示错误
- 登录错误信息

### 规范建议

1. 空状态应包含：
- 图标
- 标题
- 说明
- 可选行动按钮

2. 加载状态应优先骨架屏
- 少用纯转圈覆盖全部页面

3. 错误状态应至少包含：
- 错误说明
- 重试建议或返回路径

## 13. 后续 UI 升级方向

### 规划中

1. 统一颜色调用方式
- 继续减少页面中的硬编码色值
- 尽量统一到主题变量和语义色

2. 统一页面模板
- 列表页模板
- 详情页模板
- 仪表盘模板
- 设置页模板

3. 统一表单和弹窗体系
- 提炼更清晰的输入、选择、确认样式规范

4. 统一数据展示组件语言
- 统计卡
- 标签
- 表格
- 空态
- 骨架态

5. 为协同编辑平台预留风格升级空间
- 编辑器场景应比普通后台更沉浸
- 评论、批注、版本切换等协作组件需要更清晰视觉层级

## 结论

当前 XMT 的 UI 已经具备现代 SaaS 内容中台的基础气质，但仍处于“有风格、未完全系统化”的阶段。

后续最重要的方向不是推翻重做，而是：

1. 先提炼现有视觉规律
2. 再统一变量、卡片、表单、表格和状态组件
3. 最后围绕协同编辑场景升级整体体验
