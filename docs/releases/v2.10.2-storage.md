# XMT v2.10.2-storage

- 版本号：`v2.10.2-storage`
- 更新时间：2026-07-24
- 升级类型：数据可见性治理与作品库展示优化
- 功能模块：Creator 数据权限、抖音作品库、驾驶舱作品榜单、封面展示

本次升级遵循“修改版本号及更改相对应的系统更新说明”的统一版本管理规则。

## 【数据权限治理】

- 新增 `CreatorDataVisibility` 服务，将公开数据查看范围与账号管理范围分开解析。
- 拥有 `creator:data:view` 的 admin、director、editor、member 等系统角色，默认可读取已经完成标准化同步的公开平台账号数据，不再依赖 `creator_account_access`。
- 同步、绑定、重新授权和账号管理仍要求对应管理权限，并限定 admin、director；未删除现有 `requirePermission()` 校验。
- Creator Agent 抖音驾驶舱、作品库、趋势和同步日志，以及通用作品分析、粉丝分析和运营报告查询统一使用查看范围。

## 【作品库优化】

- `GET /api/creator-agent/douyin/works` 新增 `limit`、`cursor`、`sort` 参数；`limit` 默认 20、最大 100。
- 响应统一为 `{ items, next_cursor, has_more, page_size }`，cursor 使用作品 `publish_time` 与 `id` 标识稳定位置，不使用 offset。
- 前端作品库按页加载，并增加明确的加载状态和“下一页”入口。
- API 统一输出 `cover_url`，优先级为 `douyin_works.cover_url`、`creator_content_items.cover_url`、`creator_content_items.raw_json` 中的 `cover_url` / `cover` / `video.cover.url_list[0]` / `video.origin_cover.url_list[0]`。
- 作品库与驾驶舱 TOP5 使用统一 `ImageFallback`，支持 `loading="lazy"`，空地址、403、过期签名及网络失败时显示占位图。

## 数据库变化

- 无数据库表、字段或索引变化。
- 本阶段不生成 migration，不修改旧迁移，不连接或修改生产数据库，不执行任何 `DELETE`，不清理历史作品。

## API变化

- `GET /api/creator-agent/douyin/dashboard`
- `GET /api/creator-agent/douyin/works`
- `GET /api/creator-agent/douyin/trends`
- `GET /api/creator-agent/douyin/sync-logs`

上述查询接口改为读取公开标准化账号查看范围；作品接口的返回结构升级为 cursor 分页结构。管理接口路径不变，权限边界不变。

## 前端变化

- 作品库从全量渲染改为每次加载 20 条，并支持继续加载下一页。
- 作品库封面与驾驶舱 TOP5 封面统一使用图片降级组件。
- 图片默认懒加载，降低长列表首屏网络压力。

## 修复问题

- 修复管理员同步数据只有管理员可见、普通系统用户无法查看作品库和分析页面的问题。
- 修复作品已有真实指标但封面字段来源不一致导致空白的问题。
- 修复作品列表一次返回全部作品造成的前端加载压力。

## 风险说明

- 查看权限扩大到系统内已授权的公开标准化平台数据，符合“平台数据共享，操作权限隔离”原则；部署前应确认系统成员均属于可信内部用户。
- `sort=score` 等排序依赖账号全量真实指标计算基线，API 只返回当前页，但服务端仍需读取该账号作品指标完成评分；进一步的统计缓存属于后续性能治理，不在本提交扩大范围。
- 外部 CDN 仍可能拒绝图片请求；前端会显示占位图，不会下载或持久化图片。
- Agent 版本保持 `v2.10.2-agent`，本阶段不修改采集、上传契约、Normalizer 或历史数据。
