# XMT v2.10.2-sync

- 版本号：`v2.10.2-sync`
- 更新时间：2026-07-24
- 升级类型：服务端同步契约升级
- 功能模块：Creator Agent 同步入口、DouyinWorkInput、双表一致性、同步日志

## 【v2.10.2-sync 服务端同步契约升级】

- 服务端支持 `contract_version=2.10.2`，无版本或 `v2.10.1` payload 继续走旧兼容链路。
- v2.10.2 只读取顶层 `works` 或 `contents`，不递归 `raw_json`，不重新识别 `music`、`module`、`manifest`。
- `DouyinWorkInput` 严格要求字符串型 `aweme_id/item_id`、非空标题和对象型 `metrics`；数字型 ID 被拒绝并写入原因统计。
- 同一份合法作品集合在单个事务内写入 `creator_content_items` 与 `douyin_works`，通过 `douyin_works.content_id` 建立关系。
- `snapshot_id` 在账号范围内幂等；重复提交不新增作品、快照或同步日志。

## 数据库变化

- `douyin_works` 新增可空 `content_id`，并为非空值建立条件唯一索引。
- `douyin_sync_logs` 新增 `contract_version`、`collection_mode`、`snapshot_id`、`summary_json`。
- `summary_json` 仅保存数量和拒绝原因，不保存完整响应正文。
- 新迁移文件只增加列和索引，并以 `SELECT` 生成污染候选报告；已发布的 v2.10.1 历史迁移文件保持原执行语义，v2.10.2-sync 不执行其中的删除语句。

## API变化

- 路径保持 `POST /api/creator-agent/data-sync` 不变。
- 服务端根据解密 payload 的 `contract_version` 分流；v2.10.2 响应增加契约版本、快照 ID、采集模式、幂等状态和统计摘要。

## 前端变化

- 无前端页面或交互变化。

## 修复问题

- 修复新 Agent payload 被旧 Normalizer 再次递归识别、双表数据源不一致、数字 ID 精度风险、重复快照重复写入及同步日志缺少分层统计的问题。

## 风险说明

- `content_id` 只会在新契约写入或更新作品时建立；本阶段不回填、不删除、不改写历史 `douyin_works`。
- 条件唯一索引部署前应检查是否存在非空 `content_id` 冲突；当前历史列不存在，因此新增时初始值均为 `NULL`。
- 旧契约继续保留原 Normalizer 行为，建议在 Agent 升级完成前持续监控旧链路拒绝与作品数量。

## 开发历史说明

- 当前版本未恢复提交 `cea5434` 中删除的 `reports/` 文件。
- 本次升级没有继续删除相关文件，也没有扩大该历史提交的影响范围。
- 如需恢复这些报告文件，应经过独立评估并使用单独提交处理，不与服务端同步契约升级混合。
