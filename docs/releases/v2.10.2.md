# XMT v2.10.2-agent

- 版本号：`v2.10.2-agent`
- 更新时间：2026-07-24
- 升级类型：Agent 契约升级与稳定性修复
- 功能模块：抖音 Creator Agent、采集分页、上传契约、编辑器右键菜单

## 【抖音 Creator Agent 契约升级】

- 安全 JSON 解析：网络响应改为读取原始文本后解析，避免浏览器原生 JSON 解析提前损失整数精度。
- 超长 `aweme_id` 保真：抖音平台标识统一以字符串在采集、解析、本地存储和上传链路中传递。
- `parseWorks` 严格作品识别：只读取指定作品数组与抖音作品标识，拒绝 `music`、`manifest`、`module` 等非作品对象。
- cursor 分页：支持 `has_more`、`max_cursor`、重复 cursor、空页、最多 100 页和失败重试保护。
- `collection_mode`：上传契约支持 `discover`、`metrics_refresh`、`full_snapshot`，本版本默认完整快照。
- `snapshot_id`：每次采集生成 UUID；Agent 本地快照表按 `snapshot_id` 幂等写入。
- 上传契约升级：增加 `contract_version: 2.10.2`、采集统计与统一 `DouyinWorkInput` 字段；`contents` 上传完整合法作品集合，`knownContentIds` 仅用于新增数量统计。

### 版本

- 系统版本：`v2.10.2-agent`
- Agent 版本：`v2.10.2-agent`
- 数据契约版本：`2.10.2`

### 数据库变化

- 不修改生产数据库和服务端业务表。
- Agent 本地 SQLite 快照表增加可空 `snapshot_id TEXT` 及条件唯一索引，旧数据保持不变。
- 抖音平台 ID 继续使用 `TEXT`，不清理或改写历史数据。

### API变化

- 现有 `POST /api/creator-agent/data-sync` 路径保持不变。
- 加密数据体新增 `contract_version`、`snapshot_id`、`collection_mode`、`collection_stats`。
- `contents` 保留服务端当前兼容字段，同时增加标准化抖音作品字段。

### 前端变化

- 编辑器 ContextMenu 使用实际尺寸计算视口边界，打开期间与 BubbleMenu 互斥。
- 保留 Yjs、协作光标和 selection 同步逻辑。

### 修复问题

- 修复超长抖音 ID 精度损失、非作品对象误识别、作品列表缺少 cursor 分页和右键菜单视觉定位异常。

### 风险说明

- v2.10.2 Agent 依赖服务端兼容层识别 `contract_version`；服务端升级前应继续保留旧契约兼容能力。
- 分页受最多 100 页、重复 cursor 和空页保护限制，超过上限时需通过采集统计识别不完整快照。

### 回滚

- 回滚 Agent 应用代码即可停止发送新契约字段。
- Agent 本地新增列和条件索引可保留，旧版本会忽略这些字段；无需删除本地或生产数据。
- 服务端 Phase 3 不在本次发布范围内。

## 【编辑器交互稳定性修复】

- 修复右键菜单视觉定位异常：继续使用 `MouseEvent.clientX/clientY`，并根据菜单实际 `offsetWidth/offsetHeight` 完成视口边界计算。
- 优化 BubbleMenu 与 ContextMenu 互斥：右键菜单打开期间暂停选区 BubbleMenu，关闭后恢复。
- 保留多人协作光标能力：未修改 Yjs、collaboration cursor 或 selection 同步逻辑。
