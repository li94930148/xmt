## 生产部署步骤

以下步骤只作为预案，本轮不执行。

1. 记录当前生产 commit、PM2 状态和服务端环境变量。
2. 备份生产 SQLite 数据库。
3. 拉取代码：`git pull`。
4. 安装依赖：`npm ci`。
5. 执行检查：`npm run check`、`npm run build`。
6. 执行数据库初始化或迁移入口，确认新增表和种子幂等执行。
7. 重启后端：`pm2 restart xmt-api --update-env`。
8. 保存 PM2 状态：`pm2 save`。
9. 如有前端静态资源服务变更，执行 `systemctl reload caddy`。
10. 验证 `/api/health`、登录页、首页、日报页、复盘页、编辑器页面和成片制作详情页。
