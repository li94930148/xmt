# 中文文案与编码体检报告

## 扫描范围

- `src`
- `docs`
- `api`
- `shared`

## 扫描方式

- 编辑器/预览入口：`rg -n "RichTextEditor|EditorContent|dangerouslySetInnerHTML|editor-content-preview" src`
- 默认黑色：`rg -n "#000000|#333333|rgb\\(0, 0, 0\\)|rgb\\(51, 51, 51\\)" src`
- 乱码特征：`rg -n "锟|涔辩爜|氓|忙|莽|茅|�|鈥|銆|鐨|鍙|鏃|涓|绋|妯" src docs api shared`
- 复核方式：对命中的文件再用上下文人工判断，避免把正常中文字符误判为乱码。

## 数量结论

- 确认真实乱码文件：3 个。
- 本轮已修复：2 个。
- 未修复：1 个，仅注释层，不影响 UI。
- 误报/终端显示问题：多处，主要由 PowerShell 对无 BOM UTF-8 的显示方式导致。

## 后续建议

1. 单独安排一轮“文案与编码清理”，先处理 `src/utils/markdown.ts` 注释，再扫全仓 UI 文案。
2. 后续统一使用 UTF-8 读取工具，例如 `rg` 或编辑器，不以 PowerShell 默认 `Get-Content` 显示作为唯一判断。
3. 如要清理所有注释乱码，应和 lint 债务一起做，避免把纯注释修复混进业务改动。
