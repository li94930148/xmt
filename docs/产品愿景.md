# 产品愿景

## 文档说明

本文档用于定义 XMT 的产品方向和演进目标。

- 已实现内容：依据当前代码、页面、路由和数据库结构整理
- 规划中内容：依据现有项目定位和已建立路线图推导，不作为当前已上线能力

## 产品愿景一句话

XMT 的目标是成为内容团队从选题到复盘的统一工作平台，并逐步演进为覆盖协同创作、流程治理、资源管理和数据经营的传媒业务中台。

## 当前产品愿景

### 已实现

当前项目已经具备“内容生产中台”的基本轮廓，核心目标已经能够从代码层面被验证：

1. 用系统替代分散式流程
- 已有选题、创作、拍摄、发布、分析等主流程页面和后端接口
- 已有消息、通知、看板、日历、灵感池等协作辅助模块

2. 用角色和权限约束业务流转
- 已有用户、角色、权限和用户角色映射数据结构
- 已有权限管理页面与后端权限接口

3. 用结构化数据沉淀团队生产过程
- 已有 topics、production、shooting、publishing、analytics 等核心表
- 已有历史记录、工作流模板、审批节点、成就、活动日志等沉淀能力

### 当前定位

XMT 当前不是单一编辑器，也不是单一报表工具，而是一个面向内容团队的流程型工作平台。

它解决的是三个核心问题：

1. 内容生产是否有统一入口
2. 团队协作是否有清晰流转
3. 过程与结果是否能被数据化记录

## 中期愿景

### 规划中

在现有基础上，XMT 的中期目标应从“流程可管理”升级为“协作可增强”。

重点方向包括：

1. 从流程管理走向协同创作
- 强化多人协作编辑
- 强化批注、审阅、版本追踪
- 强化跨角色协同效率

2. 从模块集合走向统一工作台
- 把选题、创作、评论、通知、任务、审批串成一个更完整的上下文
- 减少信息在多个页面和多个模块间割裂

3. 从记录结果走向支持决策
- 不仅记录发布和分析结果
- 还要支撑团队产能、流程效率和选题效果判断

## 长期愿景

### 规划中

长期来看，XMT 应该从内容团队内部系统，演进为传媒公司级中台。

目标包括：

1. 支撑多团队、多角色、多项目协同
2. 支撑内容资产、资源、账号和组织结构统一治理
3. 支撑从生产管理到经营管理的数据闭环
4. 支撑从单体系统向平台化能力演进

## 目标用户愿景

### 已实现

当前系统已经明确服务以下用户：

1. 管理员
希望统一管理系统、权限、用户和备份

2. 编导
希望推进选题、审核流程、协调拍摄与发布

3. 内容成员
希望完成选题提交、创作执行、协作反馈

4. 编辑
希望聚焦内容编辑和脚本生产

### 规划中

随着平台演进，目标用户还可以进一步扩展到：

1. 项目负责人
关注进度、资源和团队效率

2. 运营负责人
关注账号、内容效果和数据复盘

3. 公司管理层
关注整体业务表现和经营指标

## 产品价值主张

### 已实现

当前 XMT 已经具备以下价值基础：

1. 让内容流程更可追踪
2. 让角色职责更可管理
3. 让协作信息更集中
4. 让生产数据更易沉淀

### 规划中

未来价值主张会进一步升级为：

1. 提升协同效率，而不仅是记录流程
2. 建立可复用的内容资产体系
3. 建立可衡量的组织与经营分析能力

## 愿景落地路径

### 已实现

当前已经落地的基础能力：

- 前后端同仓的完整应用骨架
- 主业务流程页面和 API
- 本地数据库与增量演进机制
- 实时消息与通知能力
- 角色权限体系
- Docker 化部署基础

### 规划中

后续愿景落地建议沿三条主线推进：

1. 业务主线
继续增强内容生产闭环的完整性和可配置性

2. 协作主线
增强编辑、批注、实时同步和流程联动

3. 平台主线
增强数据治理、资源治理和组织治理
