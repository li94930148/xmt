# 产品愿景

## 愿景一句话

把 XMT 从一个好用的内容生产中台，逐步做成我们自己的品牌中台产品，并在此基础上扩展为可服务更多行业的通用协作生产平台。

## 为什么做这件事

当前很多内容团队的问题并不是没有工具，而是：

1. 工具分散
2. 流程断裂
3. 协作留痕不完整
4. 版本与状态不统一
5. 数据复盘和组织治理弱

XMT 的意义不是简单把这些环节搬到线上，而是把“内容从想法到结果”的整个过程沉淀成可持续运转的系统能力。

## 第一阶段愿景：做好内容生产中台

在第一阶段，XMT 的核心目标是成为内容团队每天都离不开的工作中台：

1. 选题在系统里产生
2. 内容在系统里推进
3. 协作在系统里留痕
4. 状态在系统里可见
5. 数据在系统里回流

这意味着系统不仅要能“记录”，还要能真正“推动工作流”。

## 第二阶段愿景：升级为自有品牌中台

当内容生产链路稳定后，XMT 要从“项目”升级为“产品”。

这里的品牌中台，不只是换一个 Logo，而是要具备真正的平台化产品特征：

1. 可配置品牌名称、登录形象、系统信息
2. 可面向不同团队和业务线交付
3. 可沉淀标准模块、标准流程、标准角色权限
4. 可形成内部统一的品牌级操作系统

这一阶段的目标是让 XMT 成为我们自己可持续经营和迭代的品牌产品，而不是一次性的内部项目。

## 第三阶段愿景：扩展为行业通用平台

在品牌中台阶段成熟后，XMT 未来要继续抽象：

1. 把内容行业里通用的协作能力提炼出来
2. 把品牌化配置、流程配置、权限配置做成平台能力
3. 让系统不只适用于当前团队，也能适用于其他组织和行业

未来可扩展的行业方向包括但不限于：

1. 传媒内容团队
2. MCN / 短视频运营团队
3. 品牌营销内容团队
4. 企业内部知识内容团队
5. 轻量项目制协作团队

## 核心产品信念

### 1. 系统应该推动协作，而不是只做记录

XMT 不应该只是表单和表格的集合，而应该成为工作真正发生的地方。

### 2. 中台不是大而全，而是流程、角色和数据的统一入口

中台的价值在于让组织协作变得更清晰，而不是简单堆功能。

### 3. 可品牌化，才有可复制价值

如果系统只能服务一个固定团队，它就永远停留在项目形态。只有做成品牌化中台，才具备产品化和复制扩展价值。

### 4. 可抽象，才有行业化潜力

未来真正重要的不是某个页面，而是沉淀出可复用的能力：

1. 工作流
2. 权限
3. 编辑协作
4. 模板
5. 数据归档与复盘
6. 品牌与系统配置

## 未来产品形态

XMT 未来理想中的产品形态是：

1. 一个拥有品牌识别的中台产品
2. 一个可配置、可扩展、可多团队使用的平台
3. 一个能够承接内容生产、协同编辑、组织治理的系统底座
4. 一个能够继续向更多行业复制的方法论产品

## 当前阶段的判断

当前 XMT 已经不再是“从零探索期”，而是进入“治理和抽象期”：

1. 基础能力已经形成
2. 核心业务链路已经跑通
3. 现在最重要的是统一口径、稳定结构、沉淀品牌化能力

因此，接下来每一份文档、每一个模块治理动作，都应服务于同一个方向：

先把 XMT 做成我们自己的品牌中台，再把它扩展成通用行业系统。

## Workflow Engine 对产品愿景的意义

当前系统已经从“记录流程状态”进入“判断流程是否合理”的阶段。Workflow Engine v2 strict 通过 Shadow、Analytics、Decision 和 Policy Gate，将工作流从配置表单升级为可治理的规则系统。

这意味着 XMT 的中台能力进一步增强：

1. 流程不只是可配置，还可以被观测。
2. 流程不只是被执行，还可以被分析。
3. 系统不只是提示用户，还可以在 strict 模式下阻止明显不合理的流程节点。
4. 风险提示和建议流转为后续行业化模板沉淀提供数据基础。

这仍然服务于同一个长期方向：先把内部内容生产流程做稳，再把可复用的 workflow policy、模板和审计能力沉淀为平台能力。
