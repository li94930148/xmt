# 产品路线图

## 路线总览

XMT 的产品路线分为四个阶段：

1. V1：内容生产中台
2. V1.5：品牌化与协作增强
3. V2：协同编辑平台
4. V3：行业通用中台

整体路径不是跳跃式推翻重做，而是：

先把当前内容生产中台做稳，再把品牌和平台能力补齐，最后把通用能力抽象出来。

## V1 当前能力

### 定位

内容生产中台

### 已具备能力

1. 选题、创作、拍摄、发布、分析主流程
2. 用户、角色、权限基础治理
3. 站内消息与通知偏好
4. 工作流模板与审批基础
5. 灵感池、资源库、日历、成就、番茄钟等辅助模块
6. 抖音数据、导出、备份等扩展模块
7. 富文本编辑与创作详情页

### 当前重点

1. 统一文档体系
2. 稳定数据库与初始化脚本口径
3. 统一 UI 与组件规范
4. 收敛历史模块和重复实现
5. 明确品牌化方向

## V1.5 规划

### 定位

自有品牌中台 + 协作增强版

### 目标

在保留当前业务能力的基础上，把 XMT 做成我们自己的品牌型产品雏形。

### 重点建设

1. 品牌配置能力
   - 系统名称
   - 登录风格
   - 品牌信息
   - 默认视觉主题

2. 协作增强
   - 更清晰的评论和批注链路
   - 更明确的版本保存反馈
   - 更完整的流程通知
   - 更多角色协同提示

3. 管理治理
   - 模块边界更清晰
   - 页面模板更统一
   - 核心组件标准化
   - 历史结构梳理

4. 可交付性增强
   - 文档齐全
   - 配置清晰
   - 品牌统一
   - 便于内部复制部署

## V2 规划

### 定位

协同编辑平台

### 目标

从“流程中台”升级为“流程 + 文档 + 协作”的平台。

### 核心方向

1. 编辑器成为核心能力
   - 更稳定的版本体系
   - 更明确的协作标记
   - 更丰富的评论/批注/审阅能力

2. 工作流与内容协同打通
   - 流程状态与编辑状态联动
   - 审批与文档上下文联动
   - 评论、消息、操作记录统一串联

3. 前后端模块化升级
   - 前端按业务域组织
   - 后端服务层化
   - 数据结构更稳定

4. 多团队协作基础
   - 为多品牌、多团队、多空间预留模型

## V3 规划

### 定位

行业通用中台

### 目标

把 XMT 从品牌型中台，进一步抽象为可被更多行业复用的通用系统。

### 核心方向

1. 平台化配置
   - 品牌配置
   - 模块开关
   - 流程模板
   - 权限模板

2. 多组织能力
   - 多团队
   - 多业务线
   - 多项目空间

3. 行业模板化
   - 传媒行业模板
   - 品牌营销团队模板
   - 企业内容团队模板
   - 轻项目协作模板

4. 数据与治理升级
   - 更完整的业务分析
   - 更稳定的报表导出
   - 更规范的审计与备份体系

## 阶段衔接原则

1. V1 先做稳，不追求过度扩张
2. V1.5 先完成品牌化和规范化
3. V2 再强化协同编辑能力
4. V3 最后做行业化抽象

## 当前建议

当前阶段最应该做的不是再加很多新功能，而是为后面的品牌化和平台化打基础：

1. 文档统一
2. 架构统一口径
3. UI 和组件统一
4. 数据与模块边界统一

## 当前 Workflow Engine 进展

工作流能力已从“模板配置”推进到“规则引擎治理”阶段。当前真实状态为 Workflow Engine v2 strict：

1. Shadow Logging：记录 workflow policy 运行日志。
2. Analytics：提供 shadow 聚合分析。
3. Decision Engine：输出风险、建议流转和置信度。
4. Partial Control：WorkflowDesigner 支持建议流转一键应用。
5. Strict Control：审批执行和模板保存已接入 policy gate。

产品意义：

1. 工作流不再只是被动配置，而开始具备规则判断能力。
2. 系统可以对高风险流程配置给出明确修复建议。
3. 后续可继续演进为更完整的 workflow engine，但当前仍保持单体架构和现有业务链路。

后续路线：

1. 稳定 strict policy 规则口径。
2. 补齐 Workflow Engine 操作手册。
3. 评估模板版本化与审批审计增强。

## V2 实时协作编辑能力

V2 阶段新增实时协作编辑能力，优先覆盖创作管理与成片制作两个核心编辑器：

1. `ProductionDetail` 支持多人实时编辑 `production.content`。
2. `ShootingDetail` 支持多人实时编辑 `shooting.script_content`。
3. 支持在线用户、协同光标/选区、正在输入提示和断线重连恢复。
4. 协作状态只作为运行时状态存在，保存结果仍回写现有业务字段。
