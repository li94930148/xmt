# 产品路线图

## 路线总览

XMT 的演进路线建议按照以下主线推进：

1. `V1` 内容生产中台
先把内容团队日常生产流程跑通，并确保流程可追踪、角色可管理、数据可复盘。

2. `V1.5` 协作增强阶段
在现有中台基础上，补齐协同编辑、评论审阅、实时协作和流程体验。

3. `V2` 协同编辑平台
从“流程管理系统”升级为“内容协作平台”，让多人创作、评审和资产协同成为核心能力。

4. `V3` 传媒公司中台
从团队工具升级为公司级业务底座，承载跨部门、跨项目、跨账号的运营与管理。

## V1 当前能力

当前项目代码已经具备或部分具备以下能力：

### 业务主流程

- 选题创建、查看、状态流转
- 创作管理与内容编辑
- 拍摄管理
- 发布管理
- 数据分析与复盘

### 基础协作能力

- 登录认证与受保护访问
- 角色与权限管理
- 站内消息与未读提醒
- Socket.IO 实时通知
- 公告、灵感池、看板、日历、资源库等辅助模块

### 系统管理能力

- 用户管理
- 备份能力
- 工作流模板与审批节点
- 导出能力
- Docker 化部署支持

### V1 阶段目标

V1 的目标不是做成复杂平台，而是把内容生产全流程从“依赖人工沟通”升级为“依赖系统流转”。

## V1.5 规划

V1.5 建议定位为“内容生产中台向协同编辑平台过渡”的版本，重点补齐体验和协同能力。

### 重点方向

1. 强化创作协作
- 优化编辑器保存机制
- 增强评论、批注、审阅反馈
- 补齐版本对比与回滚思路

2. 强化实时同步
- 扩展实时事件覆盖范围
- 支持更多页面的协同状态刷新
- 优化消息通知与系统提示联动

3. 强化流程可配置性
- 优化工作流模板管理
- 增强节点审批规则与角色分配能力
- 让业务流程更适配不同团队

4. 强化运营视角
- 增加报表聚合视图
- 增强任务看板和日历联动
- 为负责人提供团队产能与效率观察面板

### V1.5 阶段目标

让系统从“可管理内容流程”提升到“可支撑团队协作”。

## V2 规划

V2 建议正式定位为“协同编辑平台”。

### 核心建设方向

1. 多人协同编辑
- 更完整的文档协作机制
- 多人同时编辑冲突处理
- 在线审阅与变更追踪

2. 内容资产体系
- 模板中心
- 素材与资源沉淀
- 标准脚本、话术、结构模板复用

3. 项目化协同
- 围绕项目而非单条内容组织任务
- 支持跨角色、跨阶段协作看板
- 建立任务、文档、审批、发布的统一上下文

4. 数据闭环升级
- 从单条内容分析升级到专题、项目、账号维度分析
- 加强发布效果和生产效率的关联分析
- 为内容策略提供数据支持

### V2 阶段目标

让 XMT 从流程工具成长为内容团队的协同工作平台，核心竞争力从“流程覆盖”转向“协作效率”。

## V3 规划

V3 建议定位为“传媒公司中台”。

### 核心建设方向

1. 组织级管理
- 多部门、多团队、多业务线支持
- 更细粒度权限体系
- 组织架构与岗位职责映射

2. 公司级资源中台
- 内容资产中心
- 账号矩阵管理
- 人员、设备、场地、素材等资源统一调度

3. 经营与决策中台
- 公司级经营看板
- 项目投入产出分析
- 团队效能与内容效果双维度治理

4. 平台化能力
- 对外接口与集成能力
- 更标准的服务拆分与治理
- 更稳定的部署、监控、审计与安全体系

### V3 阶段目标

让 XMT 成为传媒公司业务运转的中台底座，不仅管理内容生产，还能支撑组织协同、资源治理和经营决策。
