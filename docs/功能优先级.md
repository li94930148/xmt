# 功能优先级

## 文档说明

本文档基于当前代码真实结构整理功能优先级。

- 已实现：当前仓库中已能从页面、路由、数据结构或服务侧确认的能力
- 规划中：结合现有产品路线和架构状态建议优先补齐的能力

本文档不代表排期承诺，主要用于后续开发决策时统一判断标准。

## 优先级分层

为便于后续协作，建议使用四级优先级：

1. `P0`
核心主流程，系统不可缺少

2. `P1`
高频协作能力，直接影响团队效率

3. `P2`
增强型能力，提升治理和体验

4. `P3`
中长期能力，偏平台化和经营化

## P0 核心主流程

### 已实现

以下能力应视为当前项目最高优先级，且已在仓库中形成真实实现：

1. 登录认证与访问保护
- 依据：`/login` 页面、受保护路由、`auth.ts` 路由、JWT 与权限中间件

2. 选题管理
- 依据：`Topics.tsx`、`AddTopic.tsx`、`TopicDetail.tsx`、`api/routes/topics.ts`

3. 内容创作管理
- 依据：`Production.tsx`、`ProductionDetail.tsx`、Tiptap 编辑器组件、`api/routes/workflow.ts`

4. 拍摄管理
- 依据：`Shooting.tsx`、`ShootingDetail.tsx`、`api/routes/workflow.ts`

5. 发布管理
- 依据：`Publishing.tsx`、`PublishingDetail.tsx`、`api/routes/workflow.ts`

6. 数据分析
- 依据：`Analytics.tsx`、`api/routes/analytics.ts`

### 规划中

P0 后续仍需补强但尚不能算完全完成的方向：

1. 主流程状态流转规则进一步收敛
2. 流程间数据联动一致性增强
3. 历史记录与版本追踪统一化

## P1 高频协作能力

### 已实现

以下能力已具备基础实现，应继续保持高优先级：

1. 消息中心与未读提醒
- 依据：`Messages.tsx`、`useSocket.ts`、`messages.ts` 路由、`messages` 表

2. 角色与权限管理
- 依据：`PermissionManagement.tsx`、`roles.ts`、`permissions.ts`、相关数据库表

3. 工作流模板与审批设计
- 依据：`WorkflowDesigner.tsx`、`workflow-templates.ts` 路由、`workflow_templates` / `workflow_nodes`

4. 看板与日历
- 依据：`Kanban.tsx`、`Calendar.tsx`、`calendar.ts` 路由、`calendar_events` 表

5. 灵感池
- 依据：`Inspirations.tsx`、`inspirations.ts` 路由、相关三张灵感表

6. 资源库
- 依据：`Resources.tsx`、`resources.ts` 路由、`resources` 表

### 规划中

建议优先补强的 P1 能力：

1. 多人协作编辑能力
- 当前已有编辑器和实时基础，但尚未看到完整多人编辑机制

2. 审阅与批注链路增强
- 当前已有评论与编辑器扩展基础，但协同评审体验仍可加强

3. 通知矩阵细化
- 当前已有通知偏好表与基础消息事件，可进一步增强事件粒度

## P2 治理与体验增强

### 已实现

以下能力已落地，但更多属于增强型模块：

1. 公告板
- 依据：`announcements.ts` 路由、`announcements` 表、公告组件

2. 成就系统
- 依据：`Achievements.tsx`、`achievements.ts` 路由、相关成就表

3. 番茄钟
- 依据：`PomodoroPage.tsx`、`PomodoroTimer.tsx`、`pomodoro.ts` 路由、`pomodoro_sessions` 表

4. 备份管理
- 依据：`BackupPage.tsx`、`backup.ts` 路由、服务启动时备份逻辑

5. 导出能力
- 依据：`ExportPage.tsx`、`export.ts` 路由

6. 抖音数据模块
- 依据：`DouyinAnalytics.tsx`、`douyin.ts` 路由、真实存在的 `douyin_*` 表

### 规划中

建议作为 P2 继续推进的方向：

1. 系统设置中心化
- 当前仍有部分系统设置在本地存储中维护

2. 统一操作审计视图
- 当前已有活动日志表，但可进一步提升为治理能力

3. 模板中心前台化
- 当前后端和数据库已有模板能力，但主导航中未见独立模板页面

## P3 中长期平台能力

### 已实现基础

当前仅具备少量平台化基础，不应误判为已完成：

1. Docker 化部署基础
2. 数据沉淀基础
3. 权限模型基础

### 规划中

以下方向建议作为中长期优先级：

1. 多项目和多团队协同
2. 更完整的内容资产管理
3. 组织级资源调度
4. 经营级分析与报表
5. 更标准的服务分层和平台治理

## 优先级建议结论

建议后续阶段遵循如下开发顺序：

1. 先稳住 `P0`
保证主流程一致性、可用性和数据完整性

2. 再强化 `P1`
优先提升协作效率、权限治理和流程体验

3. 然后推进 `P2`
逐步补强运营和治理型模块

4. 最后布局 `P3`
为传媒公司中台目标做平台化准备
