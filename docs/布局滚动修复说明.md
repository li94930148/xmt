# 布局滚动修复说明

## 修复目标

- 桌面端左侧导航固定，不随主内容滚动。
- Sidebar 菜单项较多时，Sidebar 内部可独立滚动。
- 主内容区域负责页面滚动，避免混乱双滚动条。
- 成片制作详情页和创作详情页长内容可以正常滚动。
- 移动端继续使用抽屉式导航，不破坏遮罩、关闭和点击行为。

## 当前策略

- 桌面端外层 Layout 使用视口级高度和隐藏溢出。
- Sidebar 使用 `sticky top-0 h-screen`，内部菜单使用 `overflow-y-auto`。
- 主内容容器使用 `md:h-screen`、`md:min-h-0`、`md:overflow-y-auto`。
- 移动端保留 document/body 滚动和 fixed overlay 导航。
