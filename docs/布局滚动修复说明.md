# 布局滚动修复说明

## 成片制作详情页无法滚动的根因

成片制作详情页使用普通页面容器，编辑器区域又使用固定 `500px` 高度。配合全局 Layout 由 document/body 承担滚动时，页面内容、顶部栏和侧边栏的滚动职责混在一起，长内容页面容易出现主内容滚动不稳定的问题。

## 参考创作管理详情页的调整

创作管理详情页已经使用 `PageShell`、卡片容器和编辑器沉浸式布局。本轮让成片制作详情页也进入统一页面壳，并把编辑器区域改为 `min-h-[calc(100vh-22rem)]`，使内容超过一屏时交给主内容区滚动。

## Sidebar 固定策略

桌面端 Layout 改为视口级 grid：

- 外层 `md:h-screen md:overflow-hidden`
- Sidebar 使用 `sticky top-0 h-screen`
- Sidebar 内部菜单保留 `overflow-y-auto`

这样页面滚动时左侧导航不再跟随主内容上下移动，菜单过多时 Sidebar 自己滚动。

## 主内容区滚动策略

桌面端主内容容器改为：

- `md:h-screen`
- `md:min-h-0`
- `md:overflow-y-auto`

移动端仍沿用原来的 document/body 滚动和抽屉式导航，避免破坏移动端遮罩、关闭和点击行为。

## 移动端注意事项

- Sidebar 移动端仍是 fixed overlay。
- 打开移动导航时继续锁定 body 滚动。
- 主内容不增加桌面端 left margin，因为仍使用 grid 列宽管理。

## 已知风险

- 个别页面如果内部强依赖 document 滚动位置，后续可能需要改为监听主内容容器。
- 当前未做浏览器自动化截图验收，已通过 TypeScript 和构建验证结构可编译。
