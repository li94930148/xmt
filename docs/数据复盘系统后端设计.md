# 数据复盘系统后端设计

## 本轮范围

第 6 轮只实现数据复盘系统数据库与后端 API MVP，不开发复杂前端页面，不重构日报、analytics、weekly-report、权限系统或编辑器。

## 数据模型

新增表：

- `retro_templates`：复盘模板，保存分类、结构 schema 和指标绑定。
- `retrospectives`：复盘实例主表，保存范围、周期、状态、结论和版本。
- `retro_metric_snapshots`：复盘指标快照，生成后不随源数据自动变化。
- `retro_actions`：复盘行动项。
- `retro_audit_logs`：复盘轻量审计日志。

常用索引：

- `retrospectives(status, period_start, period_end)`
- `retrospectives(scope_type, scope_id)`
- `retrospectives(owner_id)`
- `retro_metric_snapshots(retro_id)`
- `retro_metric_snapshots(metric_key)`
- `retro_actions(retro_id)`
- `retro_actions(owner_id, status)`
- `retro_actions(due_date, status)`

所有建表语句使用 `CREATE TABLE IF NOT EXISTS`，不清空旧数据，不修改日报表结构。

## 权限

新增权限码：

- `analytics:retro:view`
- `analytics:retro:create`
- `analytics:retro:publish`
- `analytics:retro:archive`
- `analytics:retro:action_manage`
- `analytics:retro:template_manage`

MVP 分配：

- `admin`：全部复盘权限。
- `director`：查看、创建、发布、管理行动项。
- `member` / `editor`：查看已发布复盘，以及查看或更新自己负责的行动项。

后端服务层保留 `admin` / `director` 角色兜底，同时读取权限表，不为复盘模块重构全局权限系统。

## API

基础路径：`/api/retrospectives`

- `GET /templates`：获取 active 模板。
- `POST /templates`：创建模板，需要模板管理权限。
- `GET /`：查询复盘列表，支持 `status`、`category`、`start`、`end`、`ownerId`。
- `POST /`：创建复盘实例，需要创建权限。
- `GET /:id`：获取详情，返回主信息、模板、快照、行动项和权限信息。
- `PUT /:id`：更新 draft 复盘基础信息和结论，使用 `version` 乐观锁。
- `POST /:id/snapshot`：以 `replace` 模式生成指标快照。
- `POST /:id/publish`：发布 draft 复盘。
- `POST /:id/archive`：归档 published 复盘。
- `POST /:id/actions`：创建行动项。
- `PATCH /actions/:actionId`：更新行动项状态、结果等。

## 状态规则

复盘状态：

- `draft`：可编辑基础信息、结论、快照，可新增行动项。
- `published`：基础信息和快照只读，可新增行动项。
- `archived`：只读，默认不允许新增行动项；行动项负责人仍可更新自己的行动项进度和结果。

行动项状态：

- `todo`
- `doing`
- `done`
- `cancelled`

行动项改为 `done` 时写入 `closed_at`；从 `done` 改回其他状态时清空 `closed_at`。

## 快照规则

快照是“生成时刻”的固化指标，不是实时 dashboard。再次生成采用 MVP 的 `replace` 语义：删除当前 `retro_id` 旧快照后重新写入。

当前快照只读取系统已有真实数据：

- 选题数量：`topics`
- 创作数量：`production`
- 发布数量：`publishing`
- 日报提交数：`daily_reports`
- 日报已审核数：`daily_reports.reviewed_at`
- 日报退回数：`daily_reports.status = rejected`
- 日报风险数：`daily_reports.risk_level`
- 日报风险/阻塞分段非空数：`daily_report_items.section_key = risk`
- 日报明日计划分段非空数：`daily_report_items.section_key = tomorrow`
- 复盘行动项数量：`retro_actions`
- 复盘行动项完成数量：`retro_actions.status = done`

每条快照写入 `source_ref_json`，记录来源表、周期和查询条件。不接抖音、小红书、视频号等外部平台真实 API，也不编造外部平台数据。

## 与日报系统关系

复盘系统只读取日报表和日报分段，用于生成快照指标；不修改日报表结构，不更新日报内容，不在日报前端页面增加复盘 UI。

## 与 analytics / weekly-report 关系

旧 `analytics` 和 `weekly-report` 继续作为实时统计或报表导出能力存在。本模块定位是“结论 + 快照 + 行动项”的业务闭环，不替代旧 analytics，也不删除旧接口。

环境变量：

- `API_BASE_URL`，默认 `http://localhost:3001/api`
- `TOKEN`
- `ADMIN_TOKEN`
