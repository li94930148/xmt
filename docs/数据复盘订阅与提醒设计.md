# 数据复盘订阅与提醒设计

## 本轮实现

本轮复用现有站内消息能力，不新增订阅表，不引入外部通知渠道。

已接入的提醒事件：

- 复盘发布：通知复盘负责人、创建人、管理员/总监。
- 复盘归档：通知复盘负责人、创建人、管理员/总监。
- 行动项分配：通知行动项负责人。
- 行动项状态或结果更新：通知复盘负责人、复盘创建人、行动项创建人。

## 实现约束

- 使用 `createMessage` 写入 `messages` 并触发现有实时推送。
- 消息写入失败只记录日志，不阻断发布、归档或行动项更新。
- 不在代码或文档中写入真实 token、账号密码或第三方密钥。

## 逾期提醒预留

后续可增加服务函数扫描：

- `retro_actions.status IN ('todo', 'doing')`
- `due_date < today`
- 复盘未删除且行动项有负责人

建议未来通过统一任务调度服务触发，避免在 API 请求中做定时副作用。
