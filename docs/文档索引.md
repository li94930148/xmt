# 文档索引

## 文档目的

本文档是 XMT 项目的 docs 入口索引，用于配合 `Codex开发规范.md` 中的 Docs-first / Docs-last 闭环。

后续修改项目时，应先按任务类型定位需要阅读的文档，再检查当前代码真实状态，最后在代码变更后同步更新相关文档。

## 总览与路线

| 文档 | 适用场景 | 说明 |
|---|---|---|
| `项目说明.md` | 了解系统定位、核心模块、当前状态 | 项目级总览入口 |
| `产品愿景.md` | 判断功能是否符合长期方向 | 描述从内容中台到行业通用系统的长期目标 |
| `产品路线图.md` | 判断功能属于 V1 / V1.5 / V2 / V3 哪个阶段 | 包含当前权限治理、Workflow Engine、协作编辑等路线状态 |
| `功能优先级.md` | 判断任务优先级和是否应当前投入 | 用于避免过早泛化或无关扩张 |
| `中台系统验收清单.md` | 判断“完整新媒体中台系统”是否可被证据证明 | 按 P0 / P1 / P2 定义验收维度和证据来源 |

## 权限治理

| 文档 | 适用场景 | 说明 |
|---|---|---|
| `权限治理现状盘点.md` | 涉及角色、权限点、接口守卫、数据归属 | 当前权限真实状态入口 |
| `权限矩阵.md` | 判断 admin / director / editor / member 能做什么 | 权限点与角色矩阵 |
| `前后端权限对齐表.md` | 涉及页面、按钮、后端接口权限对齐 | 判断前端入口和后端守卫是否一致 |

当前权限治理状态：

1. 敏感只读治理 API 已完成后端权限收口。
2. 选题、创作、资源库、排期日历已冻结为数据归属优先模型。
3. 评论创建已接入 `workflow:comment`。
4. 灵感删除已统一为 owner-or-permission。
5. 选题创建已接入 `topic:create`。
6. 选题状态变更仍保留 `canAccessTopic + 状态机`，后续如拆分关键状态流转再评估 `topic:status`。

## 编辑器与协作

| 文档 | 适用场景 | 说明 |
|---|---|---|
| `编辑器运行时规则.md` | 涉及 ContentEditor、底层 Tiptap Editor、编辑器引用边界 | 编辑器运行时主入口 |
| `编辑器写入一致性规范.md` | 涉及 Yjs / database / snapshot 写入路径 | 防止多写路径和状态漂移 |
| `语义统一规范.md` | 涉及 `production.content`、`shooting.script_content`、`collaborationKey` | 内容字段和协作状态语义 |

编辑器相关任务必须优先确认：

1. 是否影响 `ContentEditor` 主入口。
2. 是否影响 `production.content` 或 `shooting.script_content`。
3. 是否影响 Yjs runtime、snapshot、writeConsistency。
4. 是否需要同步编辑器运行时或写入一致性文档。

## Workflow Engine

| 文档 | 适用场景 | 说明 |
|---|---|---|
| `系统架构.md` | Workflow Engine 当前运行状态、架构关系 | 当前为 `v2_strict` 强约束状态 |
| `数据库设计.md` | `workflow_shadow_logs`、`approval_records` 等数据语义 | 区分观测事实和业务事实 |
| `数据库语义冻结层.md` | Workflow 相关表不可混用的语义边界 | 防止把 shadow logs 当作审批事实 |
| `Codex开发规范.md` | Workflow 修改前确认事项 | 修改 policy、decision、strict gate 前必须确认影响面 |

Workflow 相关任务必须确认：

1. 是否改变状态转换规则。
2. 是否改变审批 API 阻断条件。
3. 是否改变 WorkflowDesigner 保存策略。
4. 是否改变 `topic.status` 或审批记录语义。
5. 是否需要更新 Workflow Engine 文档口径。

## 维护规则

1. 新增重要文档时，应同步更新本文档。
2. 删除、合并或重命名文档时，应同步更新本文档。
3. 如果某个任务发现文档与代码不一致，应优先修正文档口径或在任务总结中明确说明。
4. 规划内容必须标注为规划，不能写成已实现。
5. 当前状态必须来自真实代码、真实数据库脚本或已经验证的运行行为。
6. 判断阶段性能力是否完成时，应优先参考 `中台系统验收清单.md` 的证据要求。
