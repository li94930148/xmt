# 日报系统前端设计

## 页面结构

- `/daily-report` 使用三段式 Tab：我的日报、团队日报、日报归档。
- 普通用户显示“我的日报”和“日报归档”。
- 管理员、编导或具备 `report:daily:view_team` / `report:daily:review` 的用户显示“团队日报”。

## 组件结构

- `DailyReportPage.tsx`：页面状态、接口编排、Tab 切换。
- `DailyReportComposer.tsx`：个人日报填写、保存草稿、提交。
- `DailyReportAutoSummaryPanel.tsx`：自动草稿建议展示和追加。
- `DailyReportTeamBoard.tsx`：团队日报列表与审核入口。
- `DailyReportArchiveList.tsx`：个人归档查询。
- `DailyReportReviewDialog.tsx`：通过/退回审核弹窗。
- `DailyReportStatusPill.tsx`：状态和风险标签。

## 接口接入清单

- `getMyDailyReport(date)` -> `GET /api/daily-reports/me`
- `saveDailyReportDraft(payload)` -> `POST /api/daily-reports/draft`
- `submitDailyReport(id)` -> `POST /api/daily-reports/:id/submit`
- `getTeamDailyReports(params)` -> `GET /api/daily-reports/team`
- `reviewDailyReport(id, payload)` -> `POST /api/daily-reports/:id/review`
- `getDailyReportArchive(params)` -> `GET /api/daily-reports/archive`
- `generateDailyReportDraft(payload)` -> `POST /api/daily-reports/generate-draft`

## 权限显示规则

- 普通用户：填写和提交自己的日报，查询自己的归档。
- 管理员/编导：显示团队日报 Tab，可查看团队日报列表并审核。
- 前端优先使用 `usePermission`，同时用 `admin` / `director` 作为 MVP 兜底。
- 后端仍是最终权限兜底，前端隐藏不作为安全边界。

## 状态流转

- `draft`：可编辑、可保存、可提交。
- `submitted`：只读，等待审核。
- `approved`：只读。
- `rejected`：可重新编辑并提交。
- `archived`：只读。

## 自动草稿交互

- 用户点击“生成自动草稿”后调用后端 `generate-draft`。
- 面板展示真实来源记录，不展示虚构内容。
- “追加到对应分段”采用追加方式，不覆盖用户已有手写内容。
- 后端无数据时显示空状态，用户可继续手动填写。

## 团队日报审核流程

- 团队日报支持日期和状态筛选。
- 只有 `submitted` 状态显示可审核操作。
- 审核弹窗支持“审核通过”和“退回修改”，审核后刷新列表。

## 归档查询规则

- 默认查询最近 7 天。
- 页面保留开始日期和结束日期输入。
- 后端限制单次最多 31 天，前端展示友好错误。
- 本轮不做管理员按用户筛选，留给第 5 轮。

## 已知限制

- 团队日报详情暂以弹窗摘要为主，未做完整详情页。
- 自动草稿只追加文本，未做来源去重。
- 权限展示仍有 role 兜底，后续应完全接入权限系统。
- 未新增自动化测试。

## 下一轮增强建议

1. 增加团队日报详情抽屉。
2. 自动草稿按来源去重，并支持选择性追加。
3. 归档支持管理员按成员筛选。
4. 增加 API 回归测试和前端交互测试。
## 第 5 轮增强记录

- 团队日报列表增加详情入口，详情以右侧抽屉展示，不新增路由。
- 详情抽屉展示成员、日期、状态、风险、手动摘要、六个日报分段、自动来源、提交/审核时间、审核人和审核备注。
- 详情抽屉内保留审核入口，只有 `submitted` 且具备管理能力时显示。
- 自动草稿面板改为按 `topic`、`production`、`publishing`、`unknown` 分组展示。
- 自动草稿支持勾选导入、全部导入、当前页面导入标记和基础去重。
- 自动草稿导入仍然只追加到目标分段末尾，不覆盖用户手写内容。
- 归档列表增加管理员成员筛选；普通用户不展示成员筛选。
- 归档列表项显示成员，并可打开同一个详情抽屉。
- 前端增加 31 天归档范围友好校验，后端仍是最终权限与范围守卫。
