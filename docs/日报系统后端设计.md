# 日报系统后端设计

## 本轮范围

本轮完成日报系统数据库与后端 API MVP，不开发复杂前端，不做数据复盘系统，不改编辑器迁移方案，不调整生产数据库结构以外的业务表。

## 数据表

新增表：

- `daily_report_templates`：日报模板，保存默认 sections。
- `daily_reports`：日报主表，按 `user_id + report_date` 唯一。
- `daily_report_items`：日报分段条目。
- `daily_report_subscriptions`：订阅关系预留。
- `daily_report_audit_logs`：草稿保存、提交、审核、生成建议等审计记录。

关键索引：

- `daily_reports(user_id, report_date)`
- `daily_reports(report_date, status)`
- `daily_reports(team_id, report_date)`
- `daily_report_items(report_id)`
- `daily_report_subscriptions(subscriber_id)`

## 权限

新增权限码：

- `report:daily:submit`
- `report:daily:view_team`
- `report:daily:review`
- `report:daily:subscribe`
- `report:daily:archive`

MVP 分配：

- `admin`：全部日报权限。
- `director`：团队查看、审核、订阅、归档。
- `member` / `editor`：提交、查看自己归档。

当前个人草稿、提交、归档自查主要依赖登录态与归属校验；团队查看和审核接口使用权限中间件。

## API

基础路径：`/api/daily-reports`

- `GET /me?date=YYYY-MM-DD`
  - 返回当前用户指定日期日报。
  - 日期缺省为北京时间今天。
  - 无日报时返回 `report: null`、`canCreate: true` 和默认模板。

- `POST /draft`
  - 创建或更新当前用户草稿。
  - 支持空草稿。
  - 仅 `draft` / `rejected` 可保存。
  - 支持 `version` 乐观锁，保存后版本递增。
  - 同步替换 `daily_report_items`。

- `POST /:id/submit`
  - 只能提交自己的日报。
  - 仅 `draft` / `rejected` 可提交。
  - 必须有手写总结或至少一条日报内容。

- `GET /team?date=YYYY-MM-DD&status=submitted`
  - 需要 `report:daily:view_team`。
  - MVP 阶段管理员/编导可查看指定日期全量日报。
  - 返回用户名称、总结摘要和条目摘要。

- `POST /:id/review`
  - 需要 `report:daily:review`。
  - body: `{ "action": "approve" | "reject", "comment": "..." }`
  - 仅 `submitted` 可审核。

- `GET /archive?start=YYYY-MM-DD&end=YYYY-MM-DD&userId=1`
  - 普通用户只能查看自己。
  - 管理员/编导可通过 `userId` 查看他人。
  - 范围最多 31 天。

- `POST /generate-draft`
  - 只基于真实业务数据生成建议，不编造数据。
  - 当前采集来源：`topics`、`production`、`publishing` 中当前用户当天创建或更新的数据。
  - 若已有 `draft` / `rejected` 日报，只更新 `auto_summary_json`，不修改手动内容、不提交。

## 状态流转

- `draft` -> `submitted`
- `rejected` -> `draft` 或 `submitted`
- `submitted` -> `approved`
- `submitted` -> `rejected`

MVP 暂不提供显式归档状态变更接口，`archived` 状态字段保留给后续自动归档或管理操作。

## 不做事项

- 不做日报复杂前端。
- 不做复盘/分析系统。
- 不改编辑器。
- 不引入新 UI 框架或后端框架。
- 不部署、不提交 Git。

## 当前权限对应关系

- 普通用户：后端通过登录态和 `user_id` 归属限制个人日报。
- 团队查看：后端使用 `report:daily:view_team`。
- 审核：后端使用 `report:daily:review`。
- 前端：优先使用 `usePermission`，并以 `admin` / `director` 作为 MVP 兜底显示规则。
## 第 5 轮增强记录

- `GET /archive` 支持管理员/总监不传 `userId` 时查询全员日报，普通用户仍只能查询自己。
- 日报列表与详情返回 `reviewerName` / `reviewerUsername`，便于前端展示审核人。
- 自动草稿来源项增加 `meta.updatedAt`，前端可展示来源更新时间。
- 日报提交、审核通过、退回接入已有 `messages` 消息系统作为最小提醒预留。
- 消息写入失败不阻断日报提交或审核主流程。
- 未新增数据库表，未改动日报表结构。
