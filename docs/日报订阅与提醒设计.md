# 日报订阅与提醒设计

## 本轮范围

本轮不新增通知中心、不新增数据库表、不改变订阅表结构。日报提醒先复用已有 `messages` 表、`createMessage` 工具和 socket 推送能力，作为订阅提醒的最小预留。

## 已接入事件

- 日报提交：通知 `admin` / `director` 用户有新的日报待审核。
- 日报审核通过：通知日报作者审核已通过。
- 日报退回：通知日报作者日报已退回，并携带审核备注。

## 当前边界

- 当前接收人仍按角色兜底，不读取复杂订阅关系。
- 当前消息入口复用全局消息系统，不新增日报专属通知页面。
- 消息写入失败只记录日志，不阻断提交和审核主流程。

## 后续建议

1. 将 `daily_report_subscriptions` 与提醒接收人正式关联。
2. 增加“只订阅某成员 / 某团队日报”的筛选规则。
3. 增加每日未提交提醒的定时任务。
4. 在前端日报详情中展示订阅状态和提醒偏好。
