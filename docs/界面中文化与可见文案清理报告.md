# 界面中文化与可见文案清理报告

## 清理范围

本轮只清理用户可见内容，不改数据库结构，不新增业务功能，不改 API 路径、权限码、类型名或代码变量名。

已检查范围：

- 复盘中心列表页和详情页。
- 复盘组件：列表、指标快照、行动项、日报风险联动、发布归档、状态标签。
- 日报组件：详情抽屉、自动草稿、自动总结、状态标签。
- 复盘后端错误提示、导出 Markdown/HTML、消息提醒内容。
- 日报和复盘 smoke 脚本生成的测试数据。

## 修改页面

- `/retrospectives`：移除页面上的模块技术解释，不再展示 `analytics` 说明。
- `/retrospectives/:id`：导出、状态、指标快照和行动项展示使用中文映射。
- `/daily-report`：日报详情抽屉、自动草稿和自动总结面板改为中文产品文案。

## 修改组件

- `DailyReportDetailDrawer`：英文标签、按钮、来源文字改为中文。
- `DailyReportAutoDraftPanel`：标题、按钮、空状态、来源和分段文案改为中文。
- `DailyReportAutoSummaryPanel`：来源类型改为中文。
- `DailyReportStatusPill`：未知状态和未知风险不再显示 raw value。
- `RetroList`：移除 `analytics` 技术说明。
- `RetroMetricSnapshotPanel`：数据来源不再展示数据库表名和字段名。
- `RetroDailyRiskPanel`：风险等级和分段使用中文映射。
- `RetroStatusPill`：未知复盘/行动项状态显示“未知状态”。

## 后端提示与导出

- 复盘 service 中用户可见错误提示改为中文。
- 复盘 Markdown / HTML 导出中的复盘状态、行动项状态、模板类型和指标名称改为中文。
- 复盘行动项更新消息不再展示 `todo -> doing` 这类 raw enum，改为中文状态变化。

## Smoke 脚本

- `scripts/smoke-retrospectives.mjs` 后续生成：
  - `回归测试复盘-时间戳`
  - `回归测试行动项-时间戳`
  - 中文结论、描述和处理结果。
- `scripts/smoke-daily-reports.mjs` 后续生成中文日报总结、分段标题、分段内容和审核意见。

## 历史 Smoke 数据

已只针对本地开发库中明确匹配自动 smoke 前缀的数据做中文化：

- `Smoke retrospective%` 复盘标题改为 `回归测试复盘-记录ID`。
- `Smoke follow-up action` 行动项标题改为 `回归测试行动项-记录ID`。
- 相关 smoke 结论、描述和处理结果改为中文。

本操作不涉及生产数据，不涉及生产迁移，不改 schema，不删除数据。

## 允许保留的英文

以下内容属于内部实现或技术边界，允许保留：

- API 路径。
- TypeScript 类型名、变量名、函数名。
- 权限码，例如 `analytics:retro:view`。
- 数据库字段名和 SQL 内部字段。
- package.json 脚本名。
- 技术文档和部署文档中的必要术语。
- smoke/E2E 命令行日志中的 `PASS` / `FAIL`，不作为产品界面文案。
