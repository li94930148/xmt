# 编辑器迁移清单

## 当前 RichTextEditor 引用

- `src/components/ContentEditor.tsx`
  - 导入 `RichTextEditor`。
  - 当 `mode === 'legacy'` 时渲染旧版编辑器。
- `src/pages/AddTopic.tsx`
  - 当前新增选题大纲编辑显式使用 `mode="legacy"`。

## 当前 Tiptap Editor 引用

- `src/components/ContentEditor.tsx`
  - 默认 `mode="rich"` 时使用 `src/components/editor/Editor.tsx`。
- `src/components/editor/Editor.tsx`
  - 内部使用 `EditorContent`。
- 间接使用页面：
  - `src/pages/TopicDetail.tsx`
  - `src/pages/ProductionDetail.tsx`
  - `src/pages/PublishingDetail.tsx`
  - 其他通过 `ContentEditor` 且未指定 `legacy` 的页面。

## dangerouslySetInnerHTML 入口

- `src/components/editor/MarkdownPreview.tsx`
- `src/pages/AddTopic.tsx`
- `src/pages/ProductionDetail.tsx`
- `src/pages/PublishingDetail.tsx`
- `src/pages/Resources.tsx`
- `src/pages/TopicDetail.tsx`

以上入口本轮已统一接入 `normalizeLegacyEditorHtmlTheme` 或 `editor-content-preview` 兼容层，用于处理历史 HTML 默认黑/深灰正文色。

## 适合迁移到 Tiptap 的页面

- `src/pages/AddTopic.tsx`
  - 当前是唯一显式 `mode="legacy"` 的业务页面。
  - 建议下一轮优先评估迁移，因为它是新增业务入口，不应长期依赖旧编辑器。

## 暂时不建议直接迁移的位置

- `src/components/ContentEditor.tsx`
  - 需要保留 `legacy` 分支作为历史兼容层。
- `src/components/RichTextEditor.tsx`
  - 不删除，作为旧内容兼容和未迁移页面兜底。
- 历史 HTML 预览页
  - 只做主题兼容，不改变保存层和历史版本策略。

## RichTextEditor 定位

- 旧版兼容层。
- 历史内容兼容层。
- 不再用于新增业务。
- 新业务优先使用 `src/components/editor/Editor.tsx`。

## 建议迁移顺序

1. 先把 `AddTopic` 大纲编辑从 `mode="legacy"` 切到 Tiptap `mode="rich"`，验证保存格式和预览兼容。
2. 保留 `RichTextEditor` 代码至少一个版本周期，观察历史内容读取情况。
3. 把迁移后的历史 HTML 清洗逻辑继续放在预览层，不改数据库存量内容。
4. 最后再评估是否移除 `legacy` 模式；当前不建议删除。
