# 编辑器运行时规则

本文档用于冻结 XMT 当前编辑器运行时规则，避免业务页面继续绕过统一入口。

## 统一入口

`src/components/ContentEditor.tsx` 是唯一业务写入口。

业务页面如需编辑富文本内容，应优先且只能通过 `ContentEditor` 接入，不应直接依赖底层编辑器实现。

## 主编辑器内核

`src/components/editor/Editor.tsx` 是唯一主编辑器内核。

1. 当前主线基于 Tiptap。
2. `ContentEditor mode="rich"` 使用该内核。
3. 后续主线能力应围绕 Tiptap Editor 扩展。

## Legacy 编辑器

`src/components/RichTextEditor.tsx` 仅允许作为 `ContentEditor mode="legacy"` 的内部兼容实现。

1. 禁止业务页面直接引用 `RichTextEditor`。
2. 禁止新增 `RichEditor` 等旧编辑器别名。
3. 禁止新增 legacy 使用路径。
4. 不删除旧组件，但进入冻结兼容状态。

## 禁止直接引用

业务页面禁止直接引用：

1. `src/components/RichTextEditor`
2. `src/components/RichTextEditor.tsx`
3. `src/components/editor/Editor`
4. `src/components/editor/Editor.tsx`

如确需使用编辑器能力，必须通过：

```tsx
import ContentEditor from '../components/ContentEditor';
```

## 输出格式

编辑器输出必须统一为 HTML string。

1. `value` 使用 HTML string。
2. `onChange` 输出 HTML string。
3. `onSave` 传递 HTML string。
4. 不引入 JSON canonical。
5. 不引入 Markdown canonical。

## 运行时原则

1. 不修改保存逻辑。
2. 不修改接口协议。
3. 不修改数据库结构。
4. 不改变内容字段结构。
5. 不实现协同编辑。
6. 不引入新的编辑器框架。
