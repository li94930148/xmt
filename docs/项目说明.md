# 项目说明

## 项目简介

XMT 是一个面向内容团队的内容生产中台项目，用于把选题策划、内容创作、拍摄执行、发布跟踪、数据复盘和团队协作集中到一个系统中管理。项目采用前后端同仓方式开发，当前已经具备较完整的业务闭环和基础协作能力。

## 项目定位

XMT 当前定位为团队内部使用的内容生产管理平台，核心目标是把分散在聊天、表格、文档和口头流转中的内容生产流程进行结构化沉淀。

从业务层面看，项目的定位可以分为三个阶段：

1. 当前阶段是内容生产中台，重点解决选题到发布的数据化流程管理问题。
2. 中期阶段会向协同编辑平台演进，重点增强多人协作、版本管理、评论审阅和流程编排能力。
3. 长期阶段会向传媒公司级业务中台演进，承载更多部门、项目、资源、数据和经营管理能力。

## 用户角色

当前代码和数据库中可以确认的用户角色包括：

1. 管理员 `admin`
负责系统配置、用户管理、角色权限、备份和全局治理。

2. 编导 `director`
负责选题审核、流程推进、拍摄与发布环节协调，以及关键业务决策。

3. 成员 `member`
负责日常内容执行，包括选题提交、内容生产、评论协作和部分数据查看。

4. 编辑 `editor`
偏向内容创作和编辑场景，主要参与脚本编写、内容修改和生产阶段协作。

## 技术栈

- React 18
- TypeScript
- Vite 6
- React Router
- Zustand
- TanStack React Query
- Tiptap 富文本编辑器
- Socket.IO Client
- Tailwind CSS

### 后端

- Node.js
- Express
- TypeScript
- Socket.IO
- JWT 认证
- bcrypt 密码加密
- express-rate-limit

### 数据与部署

- SQLite 文件数据库，使用 `@libsql/client` 访问
- Docker / Docker Compose 部署支持
- 本地 `dist` 静态资源托管
- 可选 HTTPS 证书目录 `certs/`

## 核心模块

根据当前代码结构，项目已经形成以下核心模块：

1. 认证与登录
包括登录、鉴权、JWT 会话和受保护路由。

2. 选题管理
包括选题创建、详情、状态推进、审核与历史记录。

3. 内容创作管理
包括创作内容编辑、版本记录、Markdown/JSON/HTML 多格式存储及编辑器能力。

4. 拍摄管理
包括拍摄计划、地点、设备、拍摄状态和脚本衔接。

5. 发布管理
包括发布平台、链接、发布时间和发布阶段内容承接。

6. 数据分析
包括基础内容数据统计和抖音数据分析模块。

7. 用户与权限管理
包括用户管理、角色管理、权限管理和多角色映射。

8. 消息与通知
包括站内消息、未读数同步、实时消息提醒和通知偏好设置。

9. 协作辅助模块
包括灵感池、公告、看板、日历、番茄钟、资源库、成就系统和导出能力。

10. 工作流与审批
包括工作流模板、审批节点、审批记录以及流程设计页面。

## 当前开发状态

从当前仓库结构判断，项目处于“可用且持续扩展”的阶段，而不是纯原型阶段。

当前已确认的开发状态：

- 前后端基础框架已完成，并能以同仓形式运行。
- 路由、页面和 API 已覆盖主要业务模块。
- 数据库初始化、增量字段迁移和默认数据种子已实现。
- 权限模型已从单角色字段扩展到角色、权限、用户角色映射。
- 实时通信已用于消息和部分协作事件同步。
- 已提供 Dockerfile、docker-compose、PM2 配置和部署文档，说明项目已具备部署落地能力。

当前仍然可以看出的阶段性特征：

- 业务能力较多，但架构层仍偏单体应用。
- 路由层承担了较多业务逻辑，服务层边界还有继续清晰化的空间。
- 数据库迁移以运行时增量 `ALTER TABLE` 为主，适合现阶段快速演进，但长期需要更规范的迁移体系。
- 部分配置和个性化状态仍保存在客户端本地存储，后续可逐步中心化。

## 长期发展目标

XMT 的长期目标不是单一的内容录入工具，而是成为内容团队和传媒业务的统一协作底座。

长期目标可以概括为：

1. 建立覆盖策划、创作、拍摄、发布、复盘的标准化内容生产流程。
2. 建立多人协同编辑、评论审阅、版本追踪和消息触达的一体化协作体验。
3. 建立面向部门、项目、账号、资源和组织的权限与治理体系。
4. 建立统一的数据指标、报表和业务分析能力，支撑经营决策。
5. 最终演进为适配传媒公司场景的业务中台，承载内容、人员、资源、流程和数据的统一管理。
