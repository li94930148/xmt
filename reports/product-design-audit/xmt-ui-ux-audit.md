# XMT UI/UX 审计报告

日期：2026-06-27  
范围：登录页、首页工作台、选题管理、内容生产详情页、Workflow 审批流、消息中心、日报系统预留入口、用户与权限管理。  
证据：同目录 `01-login.png` 到 `09-permissions.png`。

## 1. 当前 UI 问题清单

1. 登录页方向高级，但品牌气质偏“黑金企业入口”，传媒行业的内容感、年轻团队活力和科技感不足。当前“岚曜 / LAN YAO”的视觉很强，但系统名仍是“新媒体协作管理系统”，品牌与产品定位没有统一。
2. 首页工作台信息堆叠较多：欢迎卡、每日一言、指标、频道数据、工具、灵感、公告都在首屏附近争夺注意力。适合展示“丰富”，但不利于用户快速判断今天该推进什么。
3. 全局导航过长，且把主流程、效率工具、数据智能、系统管理混在一条侧栏里。年轻团队的日常工作入口应该更像“内容生产驾驶舱”，而不是传统后台菜单树。
4. 选题管理列表密度高，字段较多，流程进度用一串小点表达，信息准确但解释性弱。用户很难第一眼看出“卡在哪个环节、谁要处理、下一步是什么”。
5. 内容生产详情页有编辑器能力，但首屏视觉重心被工具栏和深色内容区占据。状态、版本、协作人、审核动作都在，但缺少“创作稿件工作台”的清晰层级。
6. Workflow 审批流页面过空，只显示一个模板卡片。对于“审批流”这种高认知负担模块，缺少节点预览、运行状态、风险提示和最近使用记录。
7. 消息中心列表可用，但消息全是同质化卡片/行，缺少按“待处理、系统通知、协作动态、成就”等类型的分组。未读状态存在，但优先级和行动入口不够明确。
8. 日报系统入口没有明确出现。当前只看到“导出报告”和“本周数据报告”，与“日报系统预留入口”的产品语义不一致。
9. 用户管理和角色权限管理是典型后台表格/卡片样式，功能清楚，但与内容生产中台的年轻化语言断层较明显。
10. 视觉语言不够统一：登录页黑金极简，后台深蓝紫科技风，首页多彩卡片，管理页冷静表格。整体像多个阶段的 UI 叠加，而不是一套品牌系统。
11. 可访问性风险：深色背景上的低对比次级文字较多；大量图标按钮依赖 hover 或视觉判断；部分状态色彩依赖颜色识别。
12. 首次进入后的系统更新弹窗会遮挡核心页面，且信息偏工程发布日志。对普通内容团队用户来说干扰较强。

## 2. 推荐设计方向

推荐方向：高级年轻化的“传媒内容控制台”，关键词是“Studio OS / 内容战情室 / 轻科技 / 高密但不压迫”。

视觉建议：

1. 保留深色底，但降低纯黑面积，改成更细腻的墨蓝/炭黑底色，配一套更明亮的内容行业强调色：电蓝、青绿、珊瑚、琥珀，避免全站只靠蓝紫渐变。
2. 统一品牌：登录页、侧栏 logo、系统标题、页面主标题应围绕一个品牌名。若“岚曜”是品牌，后台不应长期显示“新媒体协作管理系统”作为主识别。
3. 把“内容状态”变成视觉主角：选题、脚本、拍摄、发布、复盘用清晰阶段条、状态胶囊、负责人头像、下一步 CTA 表达。
4. 图标风格统一使用 lucide 线性图标即可，但状态类图标可增加轻量动效和颜色语义。
5. 管理型页面减少装饰，内容型页面增加媒体行业语境：频道、平台、脚本、素材、发布计划、数据反馈。

## 3. 信息架构优化建议

建议把侧栏从“功能清单”重构为 5 个一级区：

1. 工作台：今日待办、我的内容、消息、日程。
2. 内容生产：选题、脚本/创作、成片、发布、资源。
3. 创意与排期：灵感池、看板、排期日历、番茄钟。
4. 数据与复盘：数据复盘、内容时间轴、导出报告、日报/周报。
5. 管理与治理：人员、角色权限、审批流、系统设置、备份、活动日志。

日报入口建议：

1. 在“数据与复盘”下预留“日报中心”，不要混在“导出报告”里。
2. 首页“实用工具”可以展示“今日日报”卡片：今日待提交、已提交人数、团队摘要。
3. 数据页可统一为“报告中心”，内部 tabs：日报、周报、月报、导出。

## 4. 交互动效建议

1. 页面切换：主内容区域使用轻量 fade + 8px translate，不动侧栏，减少后台切换的割裂。
2. 状态流转：选题到创作、创作到审核等节点使用短暂的进度点亮动效，突出“生产流”。
3. 列表 hover：不仅高亮行，还展示下一步动作，如“去审核”“进入创作”“查看稿件”。
4. 编辑器保存：把“正在同步/已同步”做成更清晰的状态微交互，减少用户对稿件丢失的焦虑。
5. 通知：消息中心未读进入时可使用轻微 highlight pulse，但避免长时间闪烁。
6. 审批流：模板卡片展开时显示节点连线动画；保存或校验失败时在对应节点上定位错误。
7. 首次更新弹窗：改为右上角更新提示或“版本更新抽屉”，不要阻断核心工作流。

## 5. 第一阶段最应该改的 5 个页面

1. 首页工作台：这是产品气质和任务入口的第一印象。优先把“今日要做什么”变成主线，弱化装饰和次要信息。
2. 选题管理：这是内容生产链路的源头。优先优化状态、负责人、下一步动作、批量处理和筛选体验。
3. 内容生产详情页：这是 XMT 区别于 OA 的核心。应强化“稿件编辑 + 协作 + 版本 + 审核”的创作工作台体验。
4. 登录页：品牌第一眼很重要。建议从黑金企业风转向更年轻的传媒工作室风，同时统一品牌名。
5. 侧栏/全局信息架构：不单独算页面也必须第一阶段处理，否则后续页面改得再好仍会被旧导航气质拖住。

备选第 5 页：如果必须在页面中选择，则优先改“消息中心”，因为它是协作闭环的入口。

## 6. 不建议动的高风险区域

1. Workflow strict gate 的业务规则和审批执行逻辑，不建议为了 UI 简化而替换成单纯权限码。
2. 用户与权限管理的后端权限模型，不建议第一阶段重构；可以先换信息呈现，不碰权限语义。
3. 内容编辑器底层能力、版本保存、协同同步，不建议做大改。第一阶段只改外层布局、状态表达和操作层级。
4. 数据归属模型，包括选题、创作、资源、日历的 owner/scope 逻辑，不建议跟随导航调整一起改。
5. 生产数据表结构和历史内容数据，不建议为了 UI 卡片化直接迁移。

## 截图步骤与健康度

1. `01-login.png` 登录页：视觉完整，品牌强，但气质偏黑金企业化。
2. `02-home-workbench.png` 首页工作台：内容丰富，年轻感较强，但任务主线不够明确。
3. `03-topics.png` 选题管理：功能完整，列表密度偏高，流程进度可解释性弱。
4. `04-production-detail.png` 内容生产详情页：编辑器能力强，但页面层级偏重工具而非稿件任务。
5. `05-workflow-designer.png` 审批流：可用但过空，缺少节点预览和治理信息。
6. `06-messages.png` 消息中心：列表清楚，但类型分组、优先级和行动入口不足。
7. `07-report-reserved-entry.png` 报告/日报预留：只有导出与周报语义，日报入口缺失。
8. `08-users.png` 用户管理：功能清晰，但视觉语言偏传统后台。
9. `09-permissions.png` 角色权限：结构清楚，但权限信息扫描成本高。
